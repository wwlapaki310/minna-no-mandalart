# 要件定義書

## プロジェクト概要

**プロジェクト名**: みんなのマンダラート  
**目的**: マンダラートを作成・共有できるWebアプリケーションの開発  
**ターゲット**: 目標設定や思考整理をしたい全ての人  

## ユーザーストーリー

### 作成者として

- [ ] マンダラートを作成したい
  - 9x9のグリッドに自由に入力できる
  - 入力途中で保存できる(ローカルストレージ)
  - 完成したら公開できる

- [ ] 作成したマンダラートに情報を付与したい
  - タイトルを付けられる
  - 説明文を追加できる
  - 作成者名(ニックネーム)を設定できる

- [ ] 公開設定を選択したい
  - 公開/非公開を選べる
  - 後から変更できる

### 閲覧者として

- [ ] みんなのマンダラートを見たい
  - 一覧で表示される
  - サムネイルで概要が分かる
  - クリックして詳細を見られる

- [ ] マンダラートを探したい
  - タイトルや内容で検索できる
  - 新着順・人気順で並べ替えられる

- [ ] マンダラートから学びたい
  - 他の人のマンダラートを参考にできる
  - 良いと思ったらリアクションできる(将来機能)

## 機能要件

### Phase 1: MVP(最小機能プロダクト)

#### 1. マンダラート作成機能

**優先度**: 高  
**説明**: 9x9グリッドでマンダラートを作成

- 81マスの入力フォーム
- 各マスにテキスト入力(100文字程度)
- リアルタイムプレビュー
- 入力補助(中央→サブテーマ→詳細の順で入力ガイド)

#### 2. マンダラート保存・投稿機能

**優先度**: 高  
**説明**: 作成したマンダラートを保存・公開

- タイトル・説明文の入力
- 作成者名の入力
- 公開/非公開の選択
- データベースへの保存

#### 3. マンダラート一覧表示機能

**優先度**: 高  
**説明**: 公開されたマンダラートを一覧表示

- カード形式で一覧表示
- タイトル、作成者、作成日時を表示
- ページネーション(10件ずつ)

#### 4. マンダラート詳細表示機能

**優先度**: 高  
**説明**: 個別のマンダラートを詳細表示

- 9x9グリッド全体の表示
- タイトル、説明文、作成者情報の表示
- 閲覧数のカウント

### Phase 2: 追加機能(優先度中)

- [ ] 検索機能(タイトル・内容)
- [ ] ソート機能(新着順・人気順)
- [ ] シェア機能(URL共有)
- [ ] 印刷・エクスポート機能(PDF/画像)

### Phase 3: 将来機能(優先度低)

- [ ] ユーザー認証
- [ ] マイページ(自分のマンダラート管理)
- [ ] お気に入り機能
- [ ] コメント機能
- [ ] タグ付け機能
- [ ] カテゴリ分類

## 非機能要件

### パフォーマンス

- ページ読み込み: 3秒以内
- マンダラート表示: 1秒以内
- スムーズな入力体験(遅延なし)

### セキュリティ

- XSS対策(入力のサニタイズ)
- CSRF対策
- 不適切なコンテンツの投稿対策(将来的に)

### 可用性

- アップタイム: 99%以上(Vercel/Supabaseのデフォルト)
- モバイル対応(レスポンシブデザイン)

### 保守性

- TypeScriptによる型安全性
- コンポーネント単位の設計
- ドキュメント整備

### コスト

- 運用コスト: 月額0円〜500円程度
- 初期開発コスト: 個人開発(無料)

## UI/UX要件

### デザインコンセプト

- **キーワード**: シンプル、可愛い、使いやすい
- **カラー**: 優しいパステルカラー、ピンク・水色系
- **フォント**: 読みやすいゴシック体

### レスポンシブ対応

- PC: 1920px以上
- タブレット: 768px〜1919px
- スマートフォン: 〜767px

### アクセシビリティ

- キーボード操作対応
- 十分なコントラスト比
- フォントサイズ調整可能

## データ要件

### データ保存期間

- マンダラート: 無期限(削除機能は将来実装)
- ログ: 30日間

### データバックアップ

- Supabaseの自動バックアップ機能を利用

## 成功指標(KPI)

- Phase 1完了(MVP公開): 2週間以内
- 初期ユーザー: 10人
- 投稿マンダラート数: 20件
- 閲覧数: 100回/月

## スケジュール(仮)

### Week 1: 環境構築・基本設計
- プロジェクトセットアップ
- Supabase設定
- 基本コンポーネント作成

### Week 2: MVP実装
- マンダラート作成機能
- 保存・投稿機能
- 一覧・詳細表示機能

### Week 3: テスト・デプロイ
- 動作確認
- UI調整
- 本番デプロイ
